Internetové protokoly

  • IP, IPv4, IPv6 - Internet Protocol
    • základní protokol celého internetu
    • zajišťuje přenos datagramů na základě IP adresy příjemce
      • datagram je prostě základní jednotka co se posílá po síti, fancy name
      • datagram obsahuje adresu příjemce a putuje po síti nezávisle na ostatních datagramech
    • protokol IP se stará i o převádění datagramů do nižších vrstev (ethernet, …) a zpět
    • v4 a v6 jsou verze IP, dnes stále převažuje v4. IPv4 adresa vypadá jako 192.0.2.146 (4 čísla 0-255)
  • TCP - Transmission Control Protocol
    • vytváří spolehlivé spojení mezi aplikacemi
    • doručí všechna data bez ztráty, ve správném pořadí
    • stojí “nad IP protokolem”, je víc high level, víc ho použiješ v aplikacích
  • HTTP - Hypertext Transfer Protocol
    • slouží pro přenos HTML/XML dokumentů
    • nejpoužívanější protokol, když chceš navštívit stránku tak ti jí ten server pošle způsobem, kterej je definovanej v tomhle protokolu
    • HTTP neni moc zabezbečený, má verzi HTTPS, který je šifrovaný a bezpečný
  • FTP - File Transfer Protocol
    • protokol pro přenos souborů mezi počítači
    • slouží pro upload i download
    • není šifrovaný, často se doplňuje protokolem SSH
  • WWW - World Wide Web
    • WWW ≠ internet. Internet jsou jen počítače a servery propojený kabelama tvořící síť, kde World Wide Web jsou všechny webový stránky, dokumenty, apod.
    • Jedná se o síť dokumentů a linků mezi nima, ne o síť počítačů
  • PING shell utility
    • malý program spustitelný v shellu, který slouží pro testování připojení a “dosažitelnosti” nějakého jiného počítače
    • také měří čas, za který se zpráva dostane na druhý počítač a zpátky
    • v případě že lze druhému počítači (nebo webové adrese nebo IP adrese) poslat zprávu, uvidí program bude vypisovat, že obdržel od druhého počítače nějaká data a jak dlouho to trvalo. Program ukončíme ctrl-C
    • v opačném případě často nebude dělat nic
  • TRACERT shell utility
    • vypíše kompletní cestu, kterou zpráva podstoupila na cestě k cíli
    • vypíše jména a identity zařízení skrz která šla
    • vypíše čas který to trvalo poslat a dostat zpátky
